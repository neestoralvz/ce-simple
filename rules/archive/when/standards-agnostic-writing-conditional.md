# Agnostic Writing Standards - KISS Implementation

**Authority**: Simple bias prevention through questions | **Purpose**: Eliminate bias through KISS principle application
**Updated**: 2025-07-27 16:30 (Mexico City) | **Status**: Active | **Lines**: ≤80

## When to Apply  
**IF any content creation** → ALL writing, documentation, analysis, communication

## Simple Anti-Bias Protocol (🎯 KISS)

### Three Questions (MANDATORY)
**Before writing anything, ask:**
1. **"¿Qué evidencia específica tenemos?"** - Link to actual system files:lines
2. **"¿Qué alternativas hemos considerado?"** - Document ≥2 other approaches  
3. **"¿Esto cumple con user-input/vision objectives?"** - Verify alignment

### Evidence-Only Writing (📊 BLOCKING)
**File References**: Every claim links to specific files:lines or system observations
**No Guessing**: If no evidence exists, state "No evidence found" 
**Quantifiable**: Use precise measurements (≥90%, 15 files, 3.2 seconds)
**Verifiable**: Third parties can confirm through system inspection

### Partnership Protocol Integration (🤝 MANDATORY)
**Use Existing Framework**: Leverage "Ask clarifying questions | Investigate beyond requests"
**Challenge Everything**: Use Proactive Intelligence to question assumptions
**Multiple Perspectives**: User, technical, maintenance viewpoints
**Document Alternatives**: Record rejected options with objective reasoning

## Implementation (📐 SIMPLE)

### Language Standards
**Technical English**: Industry-standard terminology with definitions
**Direct Construction**: [ACTION] + [OBJECT] + [CRITERIA] format
**Question Integration**: Include challenging questions naturally in flow

### Success Criteria (🎯 SIMPLE)
**Question Coverage**: ≥3 challenging questions asked per major section
**Evidence Density**: ≥95% statements with system references  
**Alternative Documentation**: ≥2 approaches considered for decisions
**User Alignment**: 100% consistency with user-input/ objectives

---

**KISS Truth**: Bias elimination achieved through simple questioning protocol leveraging existing Partnership Protocol framework rather than complex new systems.
